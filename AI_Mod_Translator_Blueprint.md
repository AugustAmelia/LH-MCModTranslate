# Техническое задание: AI Mod Translator (C# / .NET 8)

## Роль
Ты — Senior .NET Developer и архитектор ПО. Твоя задача — разработать высокопроизводительное Windows-приложение для автоматизированного и ручного перевода модов Minecraft.

## Технологический стек
- **Язык:** C# 12 / .NET 8
- **UI:** WPF (Windows Presentation Foundation) с использованием паттерна MVVM (CommunityToolkit.Mvvm).
- **Стиль:** Современный темный интерфейс (Material Design или Fluent UI).
- **База данных:** SQLite или LiteDB для хранения Translation Memory (TM).
- **AI Интеграция:** Semantic Kernel (для GPT/Claude) или прямые SDK (DeepL, OpenAI).
- **Парсинг:** Newtonsoft.Json / System.Text.Json.

## Основной функционал

### 1. Модуль парсинга (File Engine)
- Поддержка форматов:
    - `.json` (Minecraft 1.13+ и Forge/Fabric ресурсы).
    - `.lang` (Legacy форматы, ключ=значение).
    - `.toml` (Конфиги и современные форматы локализации).
- Сохранение структуры файлов: при записи перевода оригинальные комментарии и форматирование не должны нарушаться.

### 2. AI Перевод и Защита Данных
- **Regex Protection:** Создать фильтр, который извлекает текст, но защищает технические теги Minecraft:
    - Коды цветов: `§[0-9a-fk-or]`
    - Плейсхолдеры: `%s`, `%d`, `{0}`, `{count}`
    - Названия предметов в скобках: `[item]`
- **Batch Translation:** Асинхронная отправка строк в AI с учетом лимитов (Rate Limits) и контекста мода.

### 3. Интерфейс редактора (UI/UX)
- **DataGrid:** Таблица с колонками: Статус (иконка), Оригинал, Перевод, AI-предложение.
- **Side Panel:** Детали выбранной строки, история правок и предложения из Translation Memory.
- **Search & Replace:** С поддержкой регулярных выражений.
- **Preview:** Окно предварительного просмотра финального файла.

### 4. Translation Memory (TM)
- Глобальная база данных, которая запоминает пары "Оригинал -> Мой Перевод".
- При нахождении идентичной строки в новом моде — предлагать вариант автоматически.

## План разработки (Этапы)

### Этап 1: Инфраструктура
- Настройка проекта WPF (.NET 8).
- Создание базовой структуры папок: `Models`, `ViewModels`, `Views`, `Services`, `Helpers`.
- Реализация логики сканирования папок и определения типов файлов.

### Этап 2: Обработка файлов
- Написание универсального парсера, который читает файл в `ObservableCollection<TranslationEntry>`.
- Реализация логики "Save" с корректной кодировкой (UTF-8 без BOM).

### Этап 3: AI и API
- Реализация `TranslationService`.
- Интеграция OpenAI API / DeepL API.
- Настройка промпта для AI: "Ты профессиональный переводчик Minecraft модов. Сохраняй игровой сленг. Не переводи технические термины и теги в скобках."

### Этап 4: Полировка UI
- Добавление прогресс-баров для массовых операций.
- Реализация подсветки синтаксиса в поле редактирования.
- Виртуализация списка для работы с файлами на 10 000+ строк без лагов.

## Инструкция для генерации кода
При реализации классов:
1. Используй `async/await` для всех IO и сетевых операций.
2. Применяй Dependency Injection для сервисов.
3. Код должен быть чистым, с XML-комментариями на русском языке.
4. UI должен быть адаптивным и поддерживать High DPI.
